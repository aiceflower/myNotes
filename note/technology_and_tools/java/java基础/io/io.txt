1.BIO：socket，为阻塞式[server.accept()]，不支持客户端太大的链接
在jdk1.5之前采用线程池加上队列的形式解决这种问题。实现一种伪异步的io通信框架。
2.IO(BIO)与NIO的区别
	其本质是阻塞与非阻塞的概念
	阻塞:程序在获取网络数据的时候，如果传输数据慢，程序会一直等待，直到传输完毕。
	非阻塞：程序直接获取已经准备就绪的数据，无须等待。
	IO为同步阻塞式，NIO为同步非阻塞
	AIO（jdk1.7后升级了NIO，即NIO2.0）异步非阻塞
3.阻塞：如tcp通信
  包含头，内容，头中告诉client，server会返回10个字节的数据，那么client会一直等待这10个数据
  传输完成，如果网络差没有传输完成，client会一直等待。
