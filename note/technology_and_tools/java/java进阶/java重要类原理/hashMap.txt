1.HashMap是用来干什么的
	用来存储key-value形式的数据的。
2.HashMap的数据结构
	数组+链表
3.数组扩大机制
	数组初始容量 16
	默认加载因子 0.75
	当数组使用情况达到 容量 * 加载因子 即 16 * 0.75 =  12时进行扩容
4.链表扩大机制
	默认阀值 threshold = 8
	当链表长度超过阀值时，数据结构改变为红黑树
7.如何判断数据在HashMap中的位置
	1）n - 1 & key.hashCode  获取数组中位置,与key.hashCode % 16 差不多
	2）通过判断该位置是否有元素确定在链表中或是在红黑树中的位置
8.如何让hash尽可能的不一样
	让hashCode的低16位和高16位进行异或
9.为什么容量尽量是2的n次方
	在数组位置进行计算时保证n-1得到的结果的二进制位全为1，更加保证计算结果更分散
10.扩容时数据会重新分配
	1）原数组的位置为空
	2）不为空
		不为空下面没有了
		不为空下面是红黑树
		不为空下面是链表
